# 更新日志

本文档用于记录 Bilibili-Boost 各版本的主要变更。

## v1.3.5

### 新增

- 新增搜索结果页视频卡片底部信息行最右侧的快捷勾选框
- 搜索结果页快捷勾选框与 UP 主空间页保持一致，支持一键标记 `已看完`，再次点击后优先恢复之前进度
- 新增“已看完阈值”可配置项，可在控制面板中通过滑块与输入框联动调整自动完成阈值

### 优化

- 自动完成阈值支持在 `90% ~ 100%` 范围内按 `1%` 精度调整，默认值为 `98%`
- 历史自动完成记录会根据当前阈值重新解释显示状态，手动完成与播放结束完成不受影响

### 修复

- 修复搜索页部分视频卡片在悬停预览时，已看标签可能引起封面下移的问题
- 修复搜索页部分视频卡片的已看标签可能被压缩成过短小条的问题
- 收紧缩略图标签宿主选择逻辑，减少对 B 站原生预览层与卡片布局的干扰

### 文档

## v1.3.0

### 新增

- 新增已看记录 `导出` / `导入` 功能，可通过控制面板管理本地记录
- 导出文件使用 `JSON` 格式，仅包含插件自己的已看记录数据

### 优化

- 手动标记 `已看完` 后再次取消时，会优先恢复到标记前的观看进度
- 控制面板中的当前视频操作支持 `取消已看完`

## v1.2.0

### 新增

- 新增 `标记看过` 功能，支持自动记录视频观看进度
- 新增 `已看 XX%` / `已看完` 两种观看状态展示
- 新增视频详情页状态区域，可手动执行 `标记已看完` 和 `清除记录`
- 新增缩略图右上角已看状态徽标
- 新增 UP 主空间主页视频卡片底部快捷标记按钮
- 新增插件控制面板，可统一管理 `标记看过` 和 `合集增强` 开关

### 存储与同步

- 已看记录改为由插件本地持久化保存
- 设置项使用 `chrome.storage.local`
- 观看记录使用 `IndexedDB`
- 新增跨标签页的已看记录同步更新

### 保留

- 完整保留 `v1.1.0` 的 `合集增强` 能力

## v1.1.0

### 新增

- 首次上线 `合集增强` 功能
- 在视频页右侧播放列表区域新增 `展开标题` 按钮
- 在视频页右侧播放列表区域新增 `展开列表` 按钮
- 支持展开被省略号截断的视频标题
- 支持拉高播放列表区域，便于浏览分 P 与合集内容

### 样式

- 按钮样式贴近 B 站原生界面风格
